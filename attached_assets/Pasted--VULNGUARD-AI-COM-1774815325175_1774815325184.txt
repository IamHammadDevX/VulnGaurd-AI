LLM PROMPT ARCHITECTURE (CRITICAL)
================================================================================

**System Prompt for Claude:**
```
You are an expert Solidity security auditor with 10+ years of experience in blockchain security.

Your task: Analyze the given smart contract code and identify ALL security vulnerabilities.

For EACH vulnerability found:
1. Identify the exact type (reentrancy, overflow, etc.)
2. Find the precise line number(s)
3. Assign severity: CRITICAL | HIGH | MEDIUM | LOW
4. Explain the vulnerability in simple terms (non-technical)
5. Explain the security risk in technical terms
6. Provide a corrected code snippet

Format your response ONLY as valid JSON (no markdown, no explanations outside JSON):
{
  "contract_name": "string",
  "total_vulnerabilities": number,
  "vulnerabilities": [
    {
      "id": number,
      "type": "string (e.g., 'Reentrancy')",
      "severity": "CRITICAL|HIGH|MEDIUM|LOW",
      "line_number": number,
      "title": "string",
      "description": "Plain English explanation",
      "technical_risk": "Detailed technical explanation",
      "vulnerable_code": "string (code snippet)",
      "fixed_code": "string (corrected code)",
      "recommendation": "Best practice to prevent this"
    }
  ],
  "summary": "Overall security assessment",
  "risk_score": number (0-100)
}
```

**Examples to include in few-shot prompting:**
- Example 1: Simple reentrancy bug (famous Ethernaut level)
- Example 2: Integer overflow in token transfer
- Example 3: Access control missing require statement
- Example 4: Safe contract (0 vulnerabilities)

**User Prompt Template:**
```
Analyze this Solidity smart contract for security vulnerabilities:

\`\`\`solidity
[USER_SUBMITTED_CODE]
\`\`\`

Return ONLY valid JSON response. Find ALL vulnerabilities. Be thorough.
```

API ENDPOINT SPECIFICATIONS
================================================================================

### POST /api/scan
**Request:**
```json
{
  "code": "string (Solidity code)",
  "contractName": "string (optional)"
}
```

**Response:**
```json
{
  "success": true,
  "scanId": "uuid",
  "contract_name": "string",
  "total_vulnerabilities": number,
  "risk_score": number,
  "vulnerabilities": [...],
  "analysis_time_ms": number,
  "timestamp": "ISO string"
}
```

**Logic:**
1. Validate Solidity syntax (basic check)
2. Call Anthropic Claude with system + user prompt
3. Parse JSON response
4. Store scan result in memory (or DB if using Supabase)
5. Return formatted response

**Error Handling:**
- Invalid Solidity syntax → 400 with helpful error
- API timeout → 504 with retry option
- Rate limit → 429 with queue info

### POST /api/generate-fix
**Request:**
```json
{
  "vulnerability": {...},
  "contractCode": "string"
}
```

**Response:**
```json
{
  "success": true,
  "original_code": "string",
  "fixed_code": "string",
  "explanation": "string",
  "resources": ["url1", "url2"]
}
```

EXAMPLE VULNERABLE CONTRACT FOR TESTING
================================================================================

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract VulnerableBank {
    mapping(address => uint256) public balances;
    
    // VULNERABILITY: Reentrancy
    function withdraw(uint256 amount) public {
        require(balances[msg.sender] >= amount);
        
        (bool success, ) = msg.sender.call{value: amount}("");
        require(success);
        
        balances[msg.sender] -= amount; // WRONG ORDER!
    }
    
    // VULNERABILITY: Integer Overflow (pre-0.8.0)
    function deposit() public payable {
        balances[msg.sender] += msg.value;
    }
    
    // VULNERABILITY: Access Control
    function emergencyWithdraw() public {
        payable(msg.sender).transfer(address(this).balance);
    }
    
    // VULNERABILITY: Logic Error
    function transfer(address to, uint256 amount) public {
        require(balances[msg.sender] > amount); // Should be >=
        balances[msg.sender] -= amount;
        balances[to] += amount;
    }
}
```
